# Contribuindo para o Click Salve

Obrigado por considerar contribuir para o Click Salve! Este documento fornece guidelines para contribuições.

## 📋 Como Contribuir

1. **Fork o repositório**
   - Clique no botão "Fork" no canto superior direito

2. **Clone seu fork**
   ```bash
   git clone https://github.com/seu-usuario/click-salve.git
   cd click-salve
   ```

3. **Crie uma branch para sua feature**
   ```bash
   git checkout -b feature/sua-feature
   ```

4. **Faça suas alterações**
   - Mantenha os estilos de código consistentes
   - Use comentários claros
   - Teste suas mudanças

5. **Commit com mensagens claras**
   ```bash
   git commit -m "Descrição clara do que foi alterado"
   ```

6. **Push para sua branch**
   ```bash
   git push origin feature/sua-feature
   ```

7. **Abra um Pull Request**
   - Descreva as mudanças realizado
   - Explique por que são necessárias

## 🎯 Diretrizes de Código

### HTML
- Use HTML5 semântico
- Adicione atributos `alt` em todas as imagens
- Use atributos ARIA para acessibilidade

### CSS
- Mantenha estilos organizados por seção
- Use comentários de seção (/* ---- SEÇÃO ---- */)
- Evite estilos inline quando possível
- Follow Mobile-First approach

### JavaScript
- Use JavaScript vanilla (sem jQuery)
- Evite console.log em produção
- Use nomes de variáveis descritivos
- Adicione comentários para lógica complexa

### Nomes de Arquivos
- Use kebab-case (hifens): `componente-novo.html`
- Sem espaços em nomes de arquivo/pasta
- Minúsculas para arquivos

## ✅ Checklist Antes de Submeter PR

- [ ] Testei em navegadores modernos (Chrome, Firefox, Safari, Edge)
- [ ] Testei responsividade (mobile, tablet, desktop)
- [ ] Código segue as diretrizes acima
- [ ] Sem erros no console do navegador
- [ ] Documentação foi atualizada se necessário
- [ ] Commit messages são claras e descritivas

## 🐛 Reportando Bugs

Abra uma issue com:
- Descrição clara do bug
- Passos para reproduzir
- Navegador e versão
- Screenshots se relevante

## 💡 Sugestões de Feature

Abra uma issue descrevendo:
- O problema que seria resolvido
- Arquitetura/design proposto
- Exemplos ou mockups se aplicável
